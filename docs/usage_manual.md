# 智牧云瞳功能流程使用手册

本文用于快速理解当前项目有哪些功能、完整业务如何流转、员工移动端、管理端和后端分别承担什么职责，以及当前完成度。它是面向使用、联调和交接的总览文档，详细接口字段仍以 `docs/help.md` 和 `docs/api.md` 为准。

## 1. 系统角色和端侧分工

| 角色/系统 | 使用对象 | 主要职责 | 当前实现位置 | 完成度 |
| --- | --- | --- | --- | --- |
| Flutter 员工端 | 现场员工 | 登录、查看任务、接收任务、按栏舍上传媒体、处理未绑定媒体、上报死猪、完成任务、查看个人统计和历史 | `lib/` | 基本完成，可继续做精细化体验 |
| Admin 管理端 | 组织管理员 | 维护组织/员工/楼栋/栏舍，创建任务，查看执行进度，复核媒体，人工纠偏，查看报表和导出 PDF | `admin/` | 基本完成，偏联调工作台，部分详情可继续产品化 |
| 后端 API | 服务端 | 鉴权、组织隔离、主数据、任务流转、媒体上传和 AI 处理、死猪上报、统计报表、PDF 生成 | 外部后端，通过 `/api` 调用 | 文档已定义，前端已按文档对接 |
| AI 模型服务 | 后端依赖 | 对上传图片/视频做猪只识别、生成处理状态和结果图 | 后端桥接 | 前端只读取处理结果和状态 |

## 2. 一句话业务闭环

管理员先维护组织主数据和员工账号，然后给员工创建盘点任务；员工在移动端登录后接收任务，按楼栋和栏舍上传图片或视频，也可以先上传未绑定媒体再补绑栏舍；后端完成 AI 识别后，管理员在管理端复核媒体、人工改数并确认结果；确认后的数据进入日报、综合报告和 PDF 导出。

## 3. 端到端主流程

| 阶段 | 操作人 | 做什么 | 前端动作 | 后端动作 | 结果 |
| --- | --- | --- | --- | --- | --- |
| 1. 登录 | 管理员/员工 | 输入账号密码登录 | 调用 `POST /user/login`，保存 `token`、`id`、`orgId`、`admin` | 校验账号，签发 token，token claims 带 `empId`、`orgId` | 后续请求带 `Authorization` |
| 2. 主数据准备 | 管理员 | 维护组织、员工、楼栋、栏舍 | Admin 调用组织/员工/楼栋/栏舍接口 | 按 token 组织隔离，保存主数据 | 创建任务有可选员工和栏舍范围 |
| 3. 创建任务 | 管理员 | 给员工下发任务 | Admin 选择员工、楼栋、栏舍，调用 `POST /task/add` | 创建任务，任务状态初始通常为 `PENDING` | 员工端能查到任务 |
| 4. 接收任务 | 员工 | 开始执行任务 | Flutter 调用 `POST /task/receive/{taskId}` | 任务状态变为 `IN_PROGRESS` | 允许上传媒体 |
| 5. 上传媒体 | 员工 | 按栏舍上传图片/视频 | Flutter 调用 `POST /inventory/media/upload` | 保存媒体，去重，触发 AI 处理，返回处理状态 | 媒体挂到具体栏舍 |
| 6. 未绑定补充 | 员工 | 先传媒体，后补选栏舍 | Flutter 调用未绑定上传、列表、绑定接口 | 后端保存未绑定媒体并在绑定后归属栏舍 | 兼容弱网和现场快速采集 |
| 7. 死猪上报 | 员工 | 上报死猪数量和照片 | Flutter 调用 `POST /inventory/dead-pig`，并查询日报 | 后端保存死猪上报和图片 | 看板和报表能反映死猪数量 |
| 8. 媒体复核 | 管理员 | 审核媒体、修正数量、确认结果 | Admin 查询媒体库，确认、解锁、改数、删除 | 后端更新媒体状态和人工数量 | 已确认媒体进入正式统计 |
| 9. 完成任务 | 员工 | 确认任务做完 | Flutter 调用 `POST /task/complete/{taskId}` | 校验未绑定、处理中、异常媒体后完成任务 | 任务闭环结束 |
| 10. 报表导出 | 管理员 | 查看日报/综合报告/PDF | Admin 调用日报、综合报告和 PDF 接口 | 后端聚合已确认媒体并生成或复用 PDF | 获得正式报表和下载链接 |

### 4.3 绑定媒体上传流程

| 步骤 | 员工动作 | Flutter 做什么 | 后端做什么 |
| --- | --- | --- | --- |
| 1 | 从任务卡片点击某个栏舍 | 进入 `UploadPage`，携带 `task/building/pen` 路由参数 | 无 |
| 2 | 选择或拍摄图片/视频 | 使用 `image_picker`，本地展示预览 | 无 |
| 3 | 可选择暂存 | `TaskCache` 把本地文件复制到缓存目录 | 无 |
| 4 | 点击上传到当前栏 | 调用 `POST /inventory/media/upload`，表单含 `taskId`、`penId`、`captureTime`、`files` | 保存媒体、去重、进入 AI 处理 |
| 5 | 查看结果 | 页面展示原图/视频、输出图、缩略图、AI 数量、处理状态 | 返回 `createdItems`、`duplicateItems`、`processingStatus` |
| 6 | 刷新状态 | 重新查任务详情、媒体库、栏舍看板 | 返回最新处理状态 |

完成度：已完成主流程。视频处理完成前依赖后端 `processingStatus`，前端已按状态展示“处理中/失败/待复核/已确认”。

### 4.4 未绑定媒体流程

| 步骤 | 员工动作 | Flutter 做什么 | 后端做什么 |
| --- | --- | --- | --- |
| 1 | 点击未绑定上传图片/视频 | 调用 `POST /inventory/media/upload/unbound`，只传 `taskId` 和文件 | 保存未绑定媒体 |
| 2 | 查看未绑定媒体列表 | 调用 `GET /inventory/media/unbound?taskId=...` | 返回该任务下未绑定媒体 |
| 3 | 绑定到当前栏舍 | 调用 `PUT /inventory/media/bind`，传 `penId`、`mediaIds` | 将媒体归属到栏舍 |
| 4 | 刷新栏舍数据 | 重新查询任务详情、媒体库、看板 | 统计归入栏舍 |

完成度：基本完成。当前移动端支持查看未绑定列表和“一键全部绑定到当前栏”，还没有做逐条勾选绑定，如果现场需要精细选择，可以继续补一个选择列表。

## 6. 后端职责说明

| 领域 | 后端需要做什么 | 前端依赖点 |
| --- | --- | --- |
| 鉴权 | 登录签发 token，token claims 包含 `empId` 和 `orgId` | 前端保存 token 并放到 `Authorization` |
| 组织隔离 | 组织级接口默认按 token 中的 `orgId` 处理 | 前端不重复传 `orgId`，避免越权 |
| 主数据 | 维护组织、员工、楼栋、栏舍，并处理逻辑删除 | 管理端提供维护入口 |
| 任务 | 创建任务、任务接收、任务完成、任务详情聚合 | 员工端和管理端都依赖 `task/detail` 聚合结果 |
| 媒体 | 保存图片/视频，做去重，区分绑定和未绑定媒体 | 员工端上传，管理端复核 |
| AI 处理 | 对媒体异步识别，生成数量、输出图、缩略图、处理状态 | 前端展示 `processingStatus` 和结果路径 |
| 复核 | 支持确认、解锁、人工改数、删除媒体 | 管理端操作媒体状态 |
| 死猪 | 保存死猪上报和图片，按栏舍日期查询 | 员工端上报，管理端复核查看 |
| 统计 | 聚合栏舍看板、媒体区间汇总、日报、综合报告 | 前端只调聚合接口，不自己拼统计 |
| PDF | 生成日报/综合报告 PDF，支持缓存复用和强制重生成 | 管理端导出并打开链接 |

## 8. 推荐真实使用顺序

### 8.1 管理员第一次使用

1. 打开 Admin 管理端。
2. 用管理员账号登录。
3. 进入“主数据”。
4. 创建或确认组织信息。
5. 创建员工账号，设置 `admin=false` 给现场员工使用。
6. 创建楼栋。
7. 在楼栋下创建栏舍。
8. 进入“任务”。
9. 选择员工、楼栋、栏舍、任务时间，创建盘点任务。
10. 等员工执行后，进入“复核”查询媒体并确认。
11. 进入“报表”查看日报、综合报告并导出 PDF。

### 8.2 员工现场使用

1. 打开 Flutter 移动端。
2. 使用管理员创建的员工账号登录。
3. 在首页“任务”查看任务列表。
4. 展开任务楼栋和栏舍。
5. 点击栏舍进入上传页，系统会自动接收任务。
6. 选择图片/视频或现场拍摄。
7. 如果网络稳定，直接“上传到当前栏”。
8. 如果先采集后整理，使用“未绑定上传图片/视频”，之后再绑定到当前栏。
9. 如发现死猪，点击栏舍行的“死猪”，填写数量、备注并上传图片。
10. 返回任务页确认栏舍完成情况。
11. 所有栏舍处理完后点击“完成任务”。

### 8.3 管理员复核和出报表

1. 进入 Admin “复核”。
2. 输入栏舍 ID 和日期，查询媒体库。
3. 查看 `processingStatus`，等待处理中媒体完成。
4. 对异常 AI 数量执行人工改数。
5. 对有效媒体执行确认。
6. 对错误媒体先解锁，必要时删除。
7. 查询死猪日报，核对当天死猪上报。
8. 进入“报表”。
9. 查询日报或综合报告。
10. 默认导出 PDF 使用缓存。
11. 只有需要重新生成最新 PDF 时，才使用“重生成 PDF”。

## 10. 联调验收清单

| 验收项 | 通过标准 |
| --- | --- |
| 管理员登录 | 登录后进入管理台，刷新页面仍保持登录 |
| 主数据 | 能新增员工、楼栋、栏舍，并在创建任务下拉中出现 |
| 创建任务 | 创建后员工端 `GET /task/{employeeId}` 能查到 |
| 员工接收 | 点击栏舍进入上传页后任务状态变为进行中 |
| 绑定上传 | 上传后媒体库出现新媒体，任务详情对应栏舍状态变化 |
| AI 状态 | 视频或图片处理状态能从处理中刷新到成功或失败 |
| 未绑定上传 | 未绑定列表能查到，绑定后出现在对应栏舍媒体库 |
| 死猪上报 | 员工提交后，员工端和管理端当天日报都能查到 |
| 媒体确认 | 确认后日报/综合报告统计能纳入该媒体 |
| 人工改数 | 改数后媒体库、看板、报表能体现人工数量 |
| 完成任务 | 存在未绑定或处理中异常时后端应拦截，全部完成后可提交 |
| PDF 导出 | 返回 `accessUrl`，链接可打开 |

## 11. 后续优化建议

| 优先级 | 建议 | 原因 |
| --- | --- | --- |
| 高 | 员工端未绑定媒体支持逐条勾选绑定 | 避免误把全部未绑定媒体绑定到同一栏舍 |
| 高 | 管理端任务详情产品化 | 当前 JSON 预览适合联调，真实管理员更需要进度卡片和明细表 |
| 高 | 管理端报表产品化 | 当前能查能导出，但可读性还可以继续提升 |
| 中 | 移动端 401 自动跳登录 | token 过期时体验更完整 |
| 中 | 移动端注册流程重新定位 | 业务上员工账号更适合由管理员创建 |
| 中 | 管理端增加分页器和编辑弹窗 | 表格数据多时更易用 |
| 中 | 增加删除确认和危险操作二次确认 | 避免误删媒体和主数据 |
| 低 | 增加端到端联调脚本 | 方便每次后端更新后快速验收闭环 |
