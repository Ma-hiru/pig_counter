# 媒体上传与复核字段说明

## 1. 图片是否保存了两次

图片上传后，后端只插入一条 `pen_picture` 记录，但会保存两个 MinIO 文件：

- `picturePath`：原始上传图片，用于留存证据。
- `outputPicturePath`：AI 标注后的识别图，用于前端复核。

这是设计行为，不是重复上传，也不是重复落库。

图片上传数据库流程：

1. 上传文件先写入临时文件。
2. 调用 SSCD 做去重。
3. 未重复时调用 YOLO 计数。
4. 后端生成 AI 标注图。
5. 上传原图到 MinIO。
6. 上传 AI 标注图到 MinIO。
7. 插入一条 `pen_picture` 记录。

如果第 5、6、7 步任一步失败，已上传的对象会回滚删除。

## 2. 视频保存规则

视频上传时先保存一条 `VIDEO` 媒体记录：

- `picturePath`：原始视频。
- `outputPicturePath`：异步处理完成后的复核视频。
- `thumbnailPath`：异步处理完成后的缩略图。

视频不是重复保存图片，而是后端异步抽帧、计数、合成复核视频。

## 3. 前端复核必须使用 mediaId

前端确认、解锁、删除、人工修正都必须传媒体 ID。

可用接口：

```http
POST /api/inventory/media/confirm
POST /api/inventory/media/unlock
DELETE /api/inventory/media/{mediaId}
POST /api/inventory/media/manual-count
```

## 4. 哪些接口返回 mediaId

### 4.1 上传返回

```http
POST /api/inventory/media/upload
POST /api/inventory/media/upload/unbound
```

返回：

```json
{
  "mediaId": 9611,
  "mediaType": "IMAGE",
  "picturePath": "http://服务器IP/minio/pig-media/media/raw/...",
  "outputPicturePath": "http://服务器IP/minio/pig-media/media/output/...",
  "thumbnailPath": "http://服务器IP/minio/pig-media/media/output/..."
}
```

### 4.2 任务详情返回

```http
GET /api/task/detail/{taskId}
```

每个栏舍的最近一条媒体现在返回 `mediaId`：

```json
{
  "penId": 9201,
  "penName": "A-01栏",
  "mediaId": 9611,
  "mediaType": "IMAGE",
  "picturePath": "http://服务器IP/minio/pig-media/media/raw/...",
  "outputPicturePath": "http://服务器IP/minio/pig-media/media/output/...",
  "thumbnailPath": "http://服务器IP/minio/pig-media/media/output/...",
  "count": 9,
  "manualCount": null,
  "processingStatus": "SUCCESS",
  "status": false
}
```

### 4.3 栏舍媒体库返回

```http
GET /api/inventory/media/library?penId=9201&date=2026-04-19
```

返回项同时包含 `id` 和 `mediaId`，前端优先用 `mediaId`：

```json
{
  "id": 9611,
  "mediaId": 9611,
  "taskId": 9401,
  "penId": 9201,
  "mediaType": "IMAGE",
  "picturePath": "http://服务器IP/minio/pig-media/media/raw/...",
  "outputPicturePath": "http://服务器IP/minio/pig-media/media/output/...",
  "thumbnailPath": "http://服务器IP/minio/pig-media/media/output/...",
  "count": 9,
  "manualCount": null,
  "processingStatus": "SUCCESS",
  "status": false
}
```

### 4.4 栏舍看板返回

```http
GET /api/pen/overview?penId=9201&date=2026-04-19
```

`latestMedia` 和 `recentMedia` 内也同时包含 `id` 和 `mediaId`。

## 5. 前端复核示例

确认媒体：

```bash
curl -X POST "http://服务器IP/api/inventory/media/confirm" \
  -H "Authorization: Bearer <ADMIN_ACCESS_TOKEN>" \
  -H "Content-Type: application/json" \
  -d '{"mediaIds":[9611],"status":true}'
```

人工修正数量：

```bash
curl -X POST "http://服务器IP/api/inventory/media/manual-count" \
  -H "Authorization: Bearer <ACCESS_TOKEN>" \
  -H "Content-Type: application/json" \
  -d '{"mediaId":9611,"manualCount":10}'
```

删除未确认媒体：

```bash
curl -X DELETE "http://服务器IP/api/inventory/media/9611" \
  -H "Authorization: Bearer <ACCESS_TOKEN>"
```

## 6. 前端处理建议

- 展示图片时优先用 `outputPicturePath`，没有则用 `picturePath`。
- 展示视频时用 `thumbnailPath` 做封面，点击后播放 `outputPicturePath`，处理未完成时可播放 `picturePath`。
- 复核按钮必须绑定 `mediaId`。
- `status=true` 表示已确认，已确认媒体不能删除，必须先解锁。
